# Contributing to `app-mobile`

`app-mobile` follows the Astrea-EIP engineering handbook for workflow, review, and documentation rules.

## Required references

Read these handbook sections before opening a pull request:

- `architecture/repositories`
- `contribution/getting-started`
- `contribution/git-workflow`
- `contribution/branches`
- `contribution/pull-requests`
- `contribution/documentation`

## Local rules

- Keep mobile UI and platform concerns in this repository.
- Update local docs when mobile setup, architecture, or runtime behavior changes.
- Keep one pull request scoped to one issue.
- Run analysis, formatting, and tests before review.
